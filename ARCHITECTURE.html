Niu-MissionControl — System Architecture

Three-Tier Command System · Niumination Ecosystem · v1.0

Command Protocols

  • Label taxonomy: [Dev] [Research] [Docs] [Urgent] [Plan] [Audit] [Quick] [Maintenance]
  • Approval flow: show plan before acting
  • Progress: [Agent]: Step X of Y — every 60s max
  • No filler phrases — lead with decision

Delegation Rules

  • Builder → code implementation, bug fixes
  • Pengawas → review, research, QA
  • Arsitek → system design, architecture decisions
  • Penjaga → ops, monitoring, maintenance
  • Niu-Flow → deep audit, full codebase generation

Verification & Failures

  • Never fabricate — report failures plainly
  • Verify tool side-effects (file writes, HTTP calls)
  • Health check cron every 15 min
  • Auto-recovery: herdr restart, dashboard restart
